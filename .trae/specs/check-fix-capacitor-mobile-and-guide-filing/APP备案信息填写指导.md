## 三、后续必须完成的事项

### 1. 配置真实的极光推送 AppKey

当前构建使用的是 `dummy` 占位值。正式 APK 与发布前，必须在构建/同步命令中传入真实值：

```powershell
cd D:\study1\DeepMindMap\v2\client
$env:VITE_JPUSH_APPKEY='你的真实AppKey'
$env:JPUSH_APPKEY='你的真实AppKey'
$env:JPUSH_CHANNEL='default'
npm run build
npx cap sync android
cd android
.\gradlew.bat assembleRelease
```

或在 `client/android/local.properties` 中写入（该文件已加入 .gitignore，不会提交）：

```properties
JPUSH_APPKEY=你的真实AppKey
JPUSH_CHANNEL=default
```

### 2. 配置 Release 签名

正式 APK 必须使用 release 签名。准备 keystore 文件后，在 `client/android/local.properties` 中配置：

```properties
KEYSTORE_PATH=D\:\\key\\deepmindmap.keystore
KEYSTORE_ALIAS=deepmindmap
KEYSTORE_PASSWORD=你的密钥库密码
KEYSTORE_ALIAS_PASSWORD=你的别名密码
```

然后执行：

```powershell
cd D:\study1\DeepMindMap\v2\client\android
.\gradlew.bat assembleRelease
```

生成的 Release APK 位于：
`client/android/app/build/outputs/apk/release/app-release.apk`

### 3. 完成算法备案或安全评估

- 若主体已具备算法备案号，请在前置许可材料中上传备案号截图或回执。
- 若尚未备案，建议尽快办理；临时可上传加盖公章的《算法安全自评估报告》或《算法合规承诺书》。

### 4. 真机安装验证

建议将 Release APK 安装到至少一台 Android 真机，验证：
- 启动无白屏、无崩溃
- 工作区与画布可正常操作
- 节点长按、工具栏、返回键工作正常
- 备案信息可在侧边栏底部查看
