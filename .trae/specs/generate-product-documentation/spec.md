# 生成产品文档（比赛/教学展示版）Spec

## Why
项目需要一个面向老师和比赛评审的专业产品文档，用于展示项目价值、技术实力和创新点。现有文档分散在 docs/ 目录和 README 中，缺乏统一、面向非技术受众的高质量展示材料。

## What Changes
- 新建 `docs/product/` 目录，存放产品级文档
- 编写《DeepMindMap 产品白皮书》（Markdown 格式）
- 编写《项目介绍 PPT 大纲》（Markdown 格式，可直接转 PPT）
- 文档内容覆盖：项目背景、核心功能、技术架构、创新点、应用场景、项目成果

## Impact
- 新增文档目录：docs/product/
- 新增文件：
  - docs/product/产品白皮书.md
  - docs/product/项目介绍PPT大纲.md
- 不影响任何业务代码和运行时逻辑

## ADDED Requirements
### Requirement: 产品白皮书
The system SHALL 提供一份完整的产品白皮书，内容包含：

#### Scenario: 项目背景
- **WHEN** 读者查看白皮书第一章
- **THEN** 应清晰了解：传统线性 AI 对话的痛点（聊几轮就忘、信息零散）、思维导图工具的静态局限、本项目如何将两者融合

#### Scenario: 核心功能介绍
- **WHEN** 读者查看白皮书第二章
- **THEN** 应看到以下功能的图文说明：
  1. 节点即对话：每个节点承载完整 AI 对话历史
  2. 非线性分支：AI 智能推荐 2-4 个延伸方向，点击自动创建子节点
  3. 节点摘要：支线聊完后自动生成核心结论摘要
  4. 多工作区管理：按主题/项目组织知识
  5. AI 工具调用：AI 直接在画布上创建/扩展节点
  6. 多模型支持：用户可填自有 API Key，后端支持模型池与 fallback
  7. 文件上传与分析：支持文档上传，AI 基于文件内容对话
  8. 中英双语：完整国际化支持
  9. 管理后台：独立 Admin 系统，含数据大盘、用户管理、对话审计、消息推送
  10. 移动端支持：基于 Capacitor 的 Android 应用

#### Scenario: 技术架构
- **WHEN** 读者查看白皮书第三章
- **THEN** 应看到清晰的技术架构图描述和前后端技术栈表格

#### Scenario: 创新点与差异化
- **WHEN** 读者查看白皮书第四章
- **THEN** 应明确理解本项目与 DeepSeek/ChatGPT（线性对话）和传统思维导图工具（静态）的核心差异

#### Scenario: 应用场景
- **WHEN** 读者查看白皮书第五章
- **THEN** 应看到至少 4 个具体应用场景，配有使用流程说明

#### Scenario: 项目成果
- **WHEN** 读者查看白皮书第六章
- **THEN** 应了解项目已上线状态、访问地址、技术实现亮点

The system SHALL 提供一份可直接用于制作 PPT 的大纲文档，结构包含：

#### Scenario: PPT 结构
- **WHEN** 用户根据大纲制作 PPT
- **THEN** 应覆盖 10-15 页幻灯片的内容，每页有明确的标题、要点和演讲备注
