为什么选择思流图？

思流图不是替代 ChatGPT，而是在需要深度探索和结构化沉淀的场景下，提供比线性对话更高效的工具。

对比维度 线性 AI 聊天（ChatGPT/DeepSeek） 思流图 DeepMindMap
对话结构 线性，所有内容混在一条消息流中 树状，每个节点独立上下文，主线支线清晰
深入细节 需复制粘贴或开新窗口，打断思路 一键分叉，上下文自动继承，不打断节奏
回到主线 翻历史记录找 节点导航树一键回到上一级
知识沉淀 靠用户自己总结 一键生成节点摘要，地图即知识库
复盘复习 翻长消息列表 直接看地图结构，每个节点有摘要
分享协作 截图或导出文本 分享整张知识地图，可视化呈现
冷启动引导 无 5 个内置模板 + 新手引导 + 延伸方向按钮
AI 能力

多模型智能调度体系

思流图接入多个 AI 模型，支持智能降级与按任务路由，确保对话质量的同时控制成本。

🤖
多模型接入
默认 GLM-4-Flash，支持 DeepSeek、GPT 等
系统支持智谱 GLM-4-Flash（默认）、DeepSeek、OpenAI GPT 等多个模型服务商。通过环境变量配置 API Key 和模型优先级，后台可动态管理模型池。
🔄
智能降级链路
主模型失败自动 fallback 到备用模型
配置 fallback 链路（如 zhipu → deepseek → openai），主模型不可用时自动切换到下一个模型，用户无感知。前端实时显示降级通知。
🛠️
AI 工具调用
AI 可直接操作思维导图
通过 Function Calling，AI 可直接调用 create_node、create_relation、update_node、expand_node 等工具，实现"地图优先模式"——AI 一次性生成整张知识大纲。
⚡
优先级队列调度
对话请求优先，后台任务排队
AI 请求按优先级队列调度：用户对话为 P0 最高优先级，标题生成、摘要提炼等后台任务为 P1，确保对话体验始终流畅。
📊
智能分叉检测
规则引擎，零模型调用
输入框智能分叉提示完全基于规则引擎实现——关键词匹配、语义偏离检测、连续追问判断，零额外模型调用成本，500ms 内响应。
📝
用户自定义 API
支持用户填入自己的 API Key
高级用户可在设置中填入自己的 API Key（支持任意 OpenAI 兼容接口），优先级高于内置模型池，实现完全自主控制。
