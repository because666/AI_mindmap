## 1. 项目概述

### 1.1 项目目标
构建一个完全独立的超级管理员后台系统，用于DeepMindMap项目的运营管理、用户管理、内容审核和数据监控。

### 1.2 设计原则
- **完全独立**：与主应用解耦，独立部署，独立端口
- **极简认证**：IP白名单 + 统一密码，无复杂RBAC
- **移动端适配**：支持手机浏览器访问管理
- **安全第一**：直连主应用数据库，不暴露额外接口

### 3.2 数据模型

#### AdminIP（IP白名单）
```typescript
interface AdminIP {
  _id: string;
  ipAddress: string;           // IP地址
  nickname: string;            // 管理员昵称
  description?: string;        // 备注说明
  isFirstAdmin: boolean;       // 是否为第一个管理员
  createdAt: Date;             // 添加时间
  createdBy?: string;          // 添加者IP（第一个为null）
  lastLoginAt?: Date;          // 最后登录时间
  loginCount: number;          // 登录次数
  isActive: boolean;           // 是否有效
}

// 索引：ipAddress（唯一）
```

#### AdminConfig（系统配置）
```typescript
interface AdminConfig {
  _id: string;
  passwordHash: string;        // bcrypt加密的密码
  passwordUpdatedAt: Date;     // 密码最后修改时间
  
  // 安全设置
  loginAttempts: {             // 登录失败记录
    ipAddress: string;
    attempts: number;
    lastAttemptAt: Date;
    lockedUntil?: Date;        // 锁定至时间
  }[];
  
  // 功能开关
  features: {
    sensitiveWordCheck: boolean;   // 敏感词检测
    auditLog: boolean;             // 审计日志
    dataExport: boolean;           // 数据导出
  };
  
  updatedAt: Date;
}
```

#### AdminSession（登录会话）
```typescript
interface AdminSession {
  _id: string;
  sessionId: string;           // Session标识
  ipAddress: string;           // 登录IP
  nickname: string;            // 登录者昵称
  userAgent: string;           // 浏览器信息
  createdAt: Date;             // 创建时间
  lastActivityAt: Date;        // 最后活动时间
  isActive: boolean;           // 是否有效
}
```

### 3.3 API接口

#### 检查IP白名单
```typescript
GET /api/auth/check-ip
Response: {
  allowed: boolean;            // 是否在白名单
  isFirstVisit: boolean;       // 是否为首次访问（用于初始化）
  message?: string;            // 提示信息
}
```

#### 初始化第一个管理员
```typescript
POST /api/auth/init
Body: {
  password: string;            // 初始密码
  confirmPassword: string;     // 确认密码
}
Response: {
  success: boolean;
  message: string;
  ipAddress: string;           // 已添加的IP
}

// 限制：只有在白名单为空时才可调用
```

#### 登录
```typescript
POST /api/auth/login
Body: {
  password: string;            // 密码
}
Response: {
  success: boolean;
  needNickname: boolean;       // 是否需要设置昵称
  sessionId?: string;
  nickname?: string;
}

// 登录失败次数检查：同IP 5分钟内失败5次，锁定15分钟
```

#### 设置昵称（首次登录）
```typescript
POST /api/auth/set-nickname
Body: {
  nickname: string;            // 昵称（2-20字符）
}
Response: {
  success: boolean;
  sessionId: string;
}
```

#### 登出
```typescript
POST /api/auth/logout
Response: { success: true }
```

#### 获取当前管理员信息
```typescript
GET /api/auth/me
Response: {
  ipAddress: string;
  nickname: string;
  loginAt: string;
}
```

```typescript
interface ChatAuditItem {
```

## 6. 安全设计

### 6.1 多层防护

```
┌─────────────────────────────────────────┐
│ 第1层：网络层                           │
│ - 可配置Nginx IP白名单                   │
│ - 可选：只允许内网/VPN访问               │

├─────────────────────────────────────────┤
│ 第3层：应用层 - 密码认证                 │

│ - Session + HTTPOnly Cookie             │
├─────────────────────────────────────────┤
│ 第4层：请求层 - 限流防护                 │
│ - 登录接口：5次/5分钟                    │

├─────────────────────────────────────────┤
│ 第5层：操作层 - 敏感操作确认             │
│ - 删除用户/数据需要二次确认              │
```

### 6.3 敏感操作二次确认

```typescript
// 危险操作列表
const dangerousActions = [
  'DELETE_USER',
  'DELETE_WORKSPACE',
  'DELETE_ALL_MESSAGES',
  'BAN_USER_PERMANENT'
];

// 二次确认流程
async function confirmDangerousAction(action: string, adminIp: string) {
  // 1. 生成6位随机确认码
  const confirmCode = Math.random().toString(36).substring(2, 8).toUpperCase();
  
  // 2. 将确认码存入Redis，5分钟有效
  await redis.setex(`confirm:${adminIp}:${action}`, 300, confirmCode);
  
  // 3. 返回给前端，要求用户输入确认码
  return { needConfirm: true, confirmCodeHint: confirmCode.substring(0, 2) + '****' };
}

// 验证确认码
async function verifyConfirmCode(action: string, adminIp: string, inputCode: string) {
  const storedCode = await redis.get(`confirm:${adminIp}:${action}`);
  if (storedCode !== inputCode) {
    throw new Error('确认码错误');
  }
  await redis.del(`confirm:${adminIp}:${action}`);
  return true;
}
```
