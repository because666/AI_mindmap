## 1. 项目概述

### 1.1 功能目标
为DeepMindMap（思流图）集成极光推送服务，实现三种消息推送场景，支持Markdown格式内容和已读追踪功能。

### 1.2 推送场景
| 场景 | 发送者 | 接收者 | 触发方式 | 已读要求 |
|------|--------|--------|----------|----------|
| 场景A - 超级广播 | 软件作者（超级管理员） | 全体用户 | 后台/Web终端 | 强制已读 |
| 场景B - 工作区运营 | 系统自动 | 工作区管理员 | 定时任务触发 | 强制已读 |
| 场景C - 工作区通知 | 工作区创建者/管理员 | 工作区协作者 | 用户主动发送 | 强制已读 |

## 3. 数据库设计

### 3.1 用户设备表 (UserDevice)
```typescript
interface UserDevice {
  _id: string;                    // MongoDB ObjectId
  userId: string;                 // 关联用户ID
  registrationId: string;         // 极光SDK返回的设备标识（唯一）
  platform: 'android' | 'ios';    // 设备平台
  deviceModel?: string;           // 设备型号（可选）
  appVersion?: string;            // App版本（可选）
  createdAt: Date;                // 首次绑定时间
  updatedAt: Date;                // 最后更新时间
  isActive: boolean;              // 是否有效（卸载重装会更新）
}
```

**索引**：
- `userId`：普通索引，用于查询用户设备
- `registrationId`：唯一索引，用于设备去重

### 3.2 推送消息表 (PushMessage)
```typescript
interface PushMessage {
  _id: string;                    // MongoDB ObjectId
  
  // 基础信息
  type: 'broadcast' | 'workspace_auto' | 'workspace_manual';  // 消息类型
  title: string;                  // 推送标题（显示在通知栏）
  content: string;                // 推送内容（Markdown格式）
  summary: string;                // 摘要（列表页展示，前100字）
  
  // 发送者信息
  senderType: 'system' | 'admin' | 'user';  // 发送者类型
  senderId?: string;              // 发送者ID（系统消息可为空）
  senderName?: string;            // 发送者名称（展示用）
  
  // 目标范围
  targetType: 'all' | 'workspace_admins' | 'workspace_members' | 'specific_users';
  targetWorkspaceId?: string;     // 目标工作区ID（场景B/C）
  targetUserIds?: string[];       // 指定用户ID列表（场景A可选）
  
  // 时间信息
  createdAt: Date;                // 创建时间
  scheduledAt?: Date;             // 定时发送时间（可选）
  sentAt?: Date;                  // 实际发送时间
  expireAt: Date;                 // 过期时间（创建时间+30天）
  
  // 已读追踪（强制已读功能）
  recipients: {
    userId: string;               // 接收用户ID
    delivered: boolean;           // 是否已送达设备
    deliveredAt?: Date;           // 送达时间
    read: boolean;                // 是否已读
    readAt?: Date;                // 阅读时间
    forcedRead: boolean;          // 是否强制阅读状态
  }[];
  
  // 统计信息
  stats: {
    totalCount: number;           // 总接收人数
    deliveredCount: number;       // 已送达人数
    readCount: number;            // 已读人数
    readRate: number;             // 阅读率（百分比）
  };
  
  // 强制已读配置
  forceRead: boolean;             // 是否启用强制已读
  forceReadDeadline?: Date;       // 强制阅读截止时间
}
```

**索引**：
- `type` + `createdAt`：复合索引，用于消息列表查询
- `targetWorkspaceId`：普通索引，用于工作区消息查询
- `expireAt`：TTL索引，30天自动删除

### 3.3 用户消息状态表 (UserPushStatus)
```typescript
interface UserPushStatus {
  _id: string;
  userId: string;                 // 用户ID
  messageId: string;              // 消息ID
  
  // 状态
  delivered: boolean;             // 是否已送达
  deliveredAt?: Date;             // 送达时间
  read: boolean;                  // 是否已读
  readAt?: Date;                  // 阅读时间
  
  // 强制已读提醒
  lastRemindAt?: Date;            // 上次提醒时间（用于强制已读）
  remindCount: number;            // 提醒次数
  
  // 归档
  archived: boolean;              // 是否归档（用户清理）
  archivedAt?: Date;              // 归档时间
  
  createdAt: Date;
  updatedAt: Date;
}
```

**索引**：
- `userId` + `read` + `archived`：复合索引，用于未读消息查询
- `userId` + `createdAt`：复合索引，用于消息列表查询

### 5.4 Markdown渲染

使用安全的Markdown渲染器：
```typescript
import { marked } from 'marked';
import DOMPurify from 'dompurify';

// 允许的HTML标签（白名单）
const allowedTags = [
  'p', 'br', 'strong', 'em', 'u', 'del',
  'h1', 'h2', 'h3', 'h4', 'h5', 'h6',
  'ul', 'ol', 'li',
  'code', 'pre', 'blockquote',
  'a', 'img',
  'table', 'thead', 'tbody', 'tr', 'th', 'td'
];

const allowedAttributes = {
  'a': ['href', 'title'],
  'img': ['src', 'alt', 'title'],
  '*': ['class']
};

function renderSafeMarkdown(content: string): string {
  // 解析Markdown
  const rawHtml = marked.parse(content, {
    async: false,
    breaks: true,
    gfm: true
  });
  
  // 清理危险HTML
  return DOMPurify.sanitize(rawHtml, {
    ALLOWED_TAGS: allowedTags,
    ALLOWED_ATTR: allowedAttributes
  });
}
```

## 6. 服务端功能设计

### 6.1 极光推送服务模块

```typescript
// src/services/pushService.ts

class PushService {
  private jpushClient: any;
  
  constructor() {
    // 初始化极光客户端
    this.jpushClient = new JPush.Client({
      appKey: process.env.JPUSH_APPKEY,
      masterSecret: process.env.JPUSH_MASTER_SECRET
    });
  }
  
  /**
   * 注册设备
   */
  async registerDevice(userId: string, registrationId: string, platform: string): Promise<void> {
    // 保存或更新设备信息
    await UserDevice.findOneAndUpdate(
      { registrationId },
      {
        userId,
        registrationId,
        platform,
        isActive: true,
        updatedAt: new Date()
      },
      { upsert: true }
    );
  }
  
  /**
   * 发送广播消息（场景A）
   */
  async sendBroadcast(options: BroadcastOptions): Promise<PushMessage> {
    const {
      title,
      content,
      targetType,
      targetUserIds,
      forceRead = true
    } = options;
    
    // 确定目标用户
    let targetUsers: string[] = [];
    if (targetType === 'all') {
      const users = await User.find({ status: 'active' }).select('_id');
      targetUsers = users.map(u => u._id.toString());
    } else if (targetType === 'specific_users') {
      targetUsers = targetUserIds || [];
    }
    
    // 创建消息记录
    const message = await PushMessage.create({
      type: 'broadcast',
      title,
      content,
      summary: extractSummary(content, 100),
      senderType: 'admin',
      targetType,
      targetUserIds: targetUsers,
      forceRead,
      forceReadDeadline: forceRead ? new Date(Date.now() + 7 * 24 * 60 * 60 * 1000) : null,
      recipients: targetUsers.map(userId => ({
        userId,
        delivered: false,
        read: false
      })),
      stats: {
        totalCount: targetUsers.length,
        deliveredCount: 0,
        readCount: 0,
        readRate: 0
      },
      expireAt: new Date(Date.now() + 30 * 24 * 60 * 60 * 1000)
    });
    
    // 获取所有目标设备的Registration ID
    const devices = await UserDevice.find({
      userId: { $in: targetUsers },
      isActive: true
    });
    
    const registrationIds = devices.map(d => d.registrationId);
    
    // 批量推送（极光单次最多1000个设备）
    for (let i = 0; i < registrationIds.length; i += 1000) {
      const batch = registrationIds.slice(i, i + 1000);
      
      await this.jpushClient.push()
        .setPlatform(JPush.ALL)
        .setAudience(JPush.registration_id(batch))
        .setNotification(
          JPush.android(title, content, 1, { messageId: message._id.toString() })
        )
        .setOptions({
          apns_production: process.env.NODE_ENV === 'production'
        })
        .send();
    }
    
    // 更新发送状态
    await PushMessage.updateOne(
      { _id: message._id },
      { sentAt: new Date() }
    );
    
    return message;
  }
  
  /**
   * 发送工作区通知（场景C）
   */
  async sendWorkspaceNotification(options: WorkspaceNotificationOptions): Promise<PushMessage> {
    const { workspaceId, senderId, title, content, importance = 'normal' } = options;
    
    // 获取工作区成员（不含发送者）
    const members = await WorkspaceMember.find({
      workspaceId,
      userId: { $ne: senderId },
      notifyEnabled: { $ne: false }
    });
    
    const targetUserIds = members.map(m => m.userId.toString());
    
    // 获取发送者信息
    const sender = await User.findById(senderId);
    
    // 创建消息
    const message = await PushMessage.create({
      type: 'workspace_manual',
      title,
      content,
      summary: extractSummary(content, 100),
      senderType: 'user',
      senderId,
      senderName: sender?.nickname || '未知用户',
      targetType: 'workspace_members',
      targetWorkspaceId: workspaceId,
      targetUserIds,
      forceRead: true,
      forceReadDeadline: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000),
      recipients: targetUserIds.map(userId => ({
        userId,
        delivered: false,
        read: false
      })),
      stats: {
        totalCount: targetUserIds.length,
        deliveredCount: 0,
        readCount: 0,
        readRate: 0
      },
      expireAt: new Date(Date.now() + 30 * 24 * 60 * 60 * 1000)
    });
    
    // 推送逻辑同上...
    
    return message;
  }
  
  /**
   * 处理送达回执
   */
  async handleDeliveryReceipt(messageId: string, registrationId: string): Promise<void> {
    const device = await UserDevice.findOne({ registrationId });
    if (!device) return;
    
    await PushMessage.updateOne(
      { _id: messageId, 'recipients.userId': device.userId },
      {
        $set: {
          'recipients.$.delivered': true,
          'recipients.$.deliveredAt': new Date()
        }
      }
    );
    
    // 更新统计
    await this.updateMessageStats(messageId);
  }
  
  /**
   * 标记已读
   */
  async markAsRead(messageId: string, userId: string): Promise<void> {
    await PushMessage.updateOne(
      { _id: messageId, 'recipients.userId': userId },
      {
        $set: {
          'recipients.$.read': true,
          'recipients.$.readAt': new Date()
        }
      }
    );
    
    await this.updateMessageStats(messageId);
  }
  
  /**
   * 更新消息统计
   */
  private async updateMessageStats(messageId: string): Promise<void> {
    const message = await PushMessage.findById(messageId);
    if (!message) return;
    
    const deliveredCount = message.recipients.filter(r => r.delivered).length;
    const readCount = message.recipients.filter(r => r.read).length;
    
    await PushMessage.updateOne(
      { _id: messageId },
      {
        $set: {
          'stats.deliveredCount': deliveredCount,
          'stats.readCount': readCount,
          'stats.readRate': Math.round((readCount / message.stats.totalCount) * 100)
        }
      }
    );
  }
}

export const pushService = new PushService();
```

### 6.2 定时任务 - 工作区运营数据推送（场景B）

```typescript
// src/jobs/workspaceStatsJob.ts

import { CronJob } from 'cron';

/**
 * 每日工作区运营数据推送
 * 每天早上9点执行
 */
export const workspaceStatsJob = new CronJob(
  '0 9 * * *',  // cron表达式：每天9点
  async () => {
    console.log('[Cron] 开始生成工作区运营数据推送...');
    
    // 获取所有工作区
    const workspaces = await Workspace.find({ status: 'active' });
    
    for (const workspace of workspaces) {
      try {
        // 计算昨日数据
        const yesterday = new Date();
        yesterday.setDate(yesterday.getDate() - 1);
        yesterday.setHours(0, 0, 0, 0);
        
        const today = new Date(yesterday);
        today.setDate(today.getDate() + 1);
        
        // 统计昨日数据
        const newNodes = await Node.countDocuments({
          workspaceId: workspace._id,
          createdAt: { $gte: yesterday, $lt: today }
        });
        
        const newMessages = await Message.countDocuments({
          workspaceId: workspace._id,
          createdAt: { $gte: yesterday, $lt: today }
        });
        
        const activeMembers = await WorkspaceMember.countDocuments({
          workspaceId: workspace._id,
          lastActiveAt: { $gte: yesterday }
        });
        
        // 如果昨日无活动，跳过
        if (newNodes === 0 && newMessages === 0 && activeMembers === 0) {
          continue;
        }
        
        // 生成推送内容
        const title = `工作区"${workspace.name}"昨日动态`;
        const content = `## 昨日数据概览
        
- **新增节点**：${newNodes} 个
- **新增消息**：${newMessages} 条
- **活跃成员**：${activeMembers} 人

继续加油，让思维流动起来！ 💪`;
        
        // 获取工作区管理员
        const admins = await WorkspaceMember.find({
          workspaceId: workspace._id,
          role: { $in: ['creator', 'admin'] }
        });
        
        const adminIds = admins.map(a => a.userId.toString());
        
        // 创建并发送推送
        await pushService.sendWorkspaceAutoNotification({
          workspaceId: workspace._id.toString(),
          title,
          content,
          targetUserIds: adminIds,
          type: 'workspace_auto'
        });
        
      } catch (error) {
        console.error(`[Cron] 处理工作区 ${workspace._id} 失败:`, error);
      }
    }
    
    console.log('[Cron] 工作区运营数据推送完成');
  },
  null,  // onComplete
  true,  // start
  'Asia/Shanghai'  // 时区
);
```

## 9. 性能考虑

| 场景 | 优化策略 |
|------|----------|
| 大规模广播（>10000人） | 分批推送，每批1000人，间隔100ms |

## 10. 错误处理

| 错误场景 | 处理策略 |
|----------|----------|
| 极光API调用失败 | 重试3次，记录错误日志，标记为发送失败 |
| Registration ID无效 | 标记设备为失效，下次重新注册 |
| 用户未注册设备 | 仅保存消息记录，用户打开App时拉取 |
| 数据库写入失败 | 事务回滚，记录错误，不发送推送 |
