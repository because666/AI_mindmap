# 思流图（DeepMindMap）抖音视频拍摄指导（vibe coding 版）

> 更新日期：2026-06-27
> 定位：vibe coding 作品记录，不是产品宣传片
> 时长：25-35秒

---

## 一、视频定位

### 核心理念
**讲的是"我用AI做了个东西"的过程，而不是"这个东西有什么功能"。**

产品只是结果，过程才是内容。观众看完应该想的是"卧槽这真是AI写的？我也想试试"，而不是"哦这个工具挺有意思"。

### 视频类型
vibe coding 记录型短视频（25-35秒），真实、轻松、不正经

### 人设定位
- 不是产品经理在介绍产品
- 是一个普通人在记录自己用AI做东西的过程
- 语气：口语化、真实、像跟朋友聊天

### 目标受众
- 对 vibe coding 感兴趣的人（最大流量池）
- 大学生、学习者（学习场景共鸣）
- 想用AI做点东西但不知道怎么开始的人

### 核心叙事线
**"不会写代码，跟AI说了一句话，它帮我做了个网站。"**

### 反推销要点
- 全程不说"快来用"、"推荐给大家"
- 产品只是作为"成品"最后自然出现
- 像在记录自己的日常，不像在打广告

---

## 二、录制环境

### 硬件

| 项目 | 建议 |
|------|------|
| 电脑 | 笔记本/台式机，确保录屏流畅 |
| 录屏软件 | OBS Studio / 系统自带录屏 |
| 浏览器 | Chrome 或 Edge |

### 录制设置
- 分辨率：1920x1080
- 浏览器全屏（F11），隐藏书签栏（Ctrl+Shift+B），缩放100%（Ctrl+0）
- 录制完成后裁剪为 9:16 竖屏（抖音）

### 配音方式
- 剪映AI配音，选口语化音色（不要太播音腔）
- 或者自己真人配音（更真实，更推荐）

## 三、分镜脚本（5个镜头，约30秒）

---

### 镜头1：钩子（0-3秒）

**画面：** DeepMindMap 成品画布的全景（先给结果，制造悬念）

**文字叠加：** "不会写代码，用AI做了个网站"（大字，屏幕中央）

**配音：** 无（纯文字 + BGM）

**拍摄要点：**
- 用已有的知识地图数据，缩小到全景
- 画面要好看，节点分布均匀
- 这3秒决定用户是否停留，文字要大要醒目

**设计意图：**
- "不会写代码"：反差感，制造好奇
- 先给结果再讲过程：悬念驱动观看

---

### 镜头2：痛点（3-8秒）

**画面：** ChatGPT/DeepSeek 的混乱对话，快速滚动

**配音：** "和AI聊复杂问题，聊几轮就忘了在问啥"

**文字叠加：** "每次都是这样"（配合配音出现）

**拍摄要点：**
- 用之前准备好的 ChatGPT 对话素材（高数学习场景）
- 快速滚动，展示对话越来越长、越来越乱
- 不需要看清具体内容，只要传达"乱"的感觉
- 5秒内完成

**设计意图：**
- "聊几轮就忘了在问啥"：直击痛点，引发共鸣
- 为后面的"所以我做了个东西"做铺垫

---

### 镜头3：转折——vibe coding 开始（8-12秒）

**画面：** 切到 TRAE 编辑器界面

**配音：** "所以我跟AI说：帮我做一个对话自动变成思维导图的工具"

**文字叠加：** "vibe coding，从一句话开始"

**拍摄要点：**
- TRAE 编辑器界面，左侧文件树，右侧代码
- 可以在对话框中输入那句话（提前打好，录制时展示即可）
- 也可以直接展示 AI 正在写代码的过程
- 4秒，节奏要快

**设计意图：**
- "跟AI说了一句话"：强调门槛极低
- "vibe coding"：蹭热点标签，吸引目标受众
- 从"痛点"到"解决方案"的转折点

---

### 镜头4：过程快剪（12-22秒）

**画面：** TRAE编辑器 + 终端 + 浏览器，快速切换（每个画面1-2秒）

**配音：** "AI写代码，我提需求，调试，部署，上线"（节奏快，像报菜名）

**画面切换顺序：**

| 时间 | 画面 | 时长 |
|------|------|------|
| 12-14秒 | TRAE中AI在生成代码（代码快速滚动） | 2秒 |
| 14-15秒 | 终端 `npm run build` 成功 | 1秒 |
| 15-17秒 | 浏览器中产品页面第一次出现 | 2秒 |
| 17-19秒 | TRAE中继续改代码（AI回复中） | 2秒 |
| 19-20秒 | 终端部署命令执行 | 1秒 |
| 20-22秒 | 浏览器中产品新功能出现 | 2秒 |

**拍摄要点：**
- **这是全视频最有"vibe coding"感的部分**
- 每个画面停留1-2秒，不要超过2秒
- 画面之间用硬切，不要花哨转场
- 终端画面要有 `npm run build`、`scp`、`pm2 restart` 这类命令
- 浏览器画面展示产品从"空白"到"有内容"的变化
- 配音节奏要跟画面切换对齐

**素材录制方法：**
- TRAE 编辑器：打开一个之前 AI 写代码的对话，录屏展示 AI 回复代码的过程
- 终端：录制一次 `npm run build` 的过程（提前确保构建成功）
- 浏览器：录制从空白页面到产品页面加载完成的过程

**设计意图：**
- 快剪传达"过程很流畅"的感觉
- "AI写代码，我提需求"：一句话概括 vibe coding
- 不解释细节，只展示节奏感

---

### 镜头5：成品展示 + 结尾（22-30秒）

**画面：** DeepMindMap 的操作演示（快节奏）

**配音：** "现在它长这样，免费的，链接评论区"

**画面操作顺序：**

| 时间 | 操作 | 时长 |
|------|------|------|
| 22-25秒 | 点击延伸方向按钮，子节点自动创建 | 3秒 |
| 25-27秒 | 点击"生成摘要"，摘要出现 | 2秒 |
| 27-29秒 | 点击导航树，一键回到主线 | 2秒 |
| 29-30秒 | 缩小到全景，展示完整知识树 | 1秒 |

**文字叠加：** "思流图 DeepMindMap · deepmindmap.work"（最后1秒出现）

**拍摄要点：**
- 功能展示要快，每个操作2-3秒，不要拖
- 不需要解释功能原理，只要展示"它能做什么"
- 最后缩小到全景，给一个视觉冲击
- 配音轻松随意，不像在介绍产品

**设计意图：**
- "现在它长这样"：产品是"成品"自然出现，不是刻意介绍
- "免费的"：消除顾虑
- "链接评论区"：引导行动，但不强推
