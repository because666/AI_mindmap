# DeepMindMap（思流图）项目介绍 PPT 大纲

> **用途**：比赛答辩 / 课程汇报 / 项目展示  
> **建议页数**：12 页  
> **演讲时长**：8-10 分钟  
> **目标受众**：老师、比赛评审、非技术背景听众

---

## 第 1 页：封面

**标题**：DeepMindMap（思流图）— AI 驱动的非线性思维导图

**副标题**：把一次深度 AI 探索，变成一张可管理、可沉淀、可分享的思维地图

**页面要素**：
- 产品 Logo（如有）
- 项目 Slogan
- 汇报人信息（姓名、学校/单位）
- 日期

**演讲备注**：
> "各位老师/评委好，今天我介绍的项目是 DeepMindMap，中文名思流图。它的核心理念是：让 AI 对话不再是一条越走越乱的线，而是一张越用越清晰的知识地图。"

---

## 第 2 页：痛点引入

**标题**：我们和 AI 聊复杂问题时，遇到了什么麻烦？

**要点**：
- 聊几轮就忘了最初在问什么
- 想回溯 earlier 信息，要翻很久
- 聊完后得到一大段文字，难以整理和复习
- 传统思维导图又是静态的，需要手动输入

**配图建议**：
- 左半：ChatGPT 长对话滚动截图（示意信息淹没）
- 右半：空白思维导图（示意静态、无内容）

**演讲备注**：
> "大家一定有过这种体验：跟 ChatGPT 或 DeepSeek 聊一个复杂话题，比如 Transformer 架构，聊了几轮之后，前面的内容已经被淹没，想找一个之前的点非常麻烦。聊完之后，你得到的是一段很长的文字，想要整理成结构化的笔记，还得自己手动提炼。"
> "另一方面，传统思维导图工具虽然结构清晰，但它是静态的，所有内容都要你自己手动输入，无法和 AI 的动态生成能力结合。"

---

## 第 3 页：市场分析

**标题**：现有方案的局限

**要点**（表格形式）：

| 产品类型 | 优势 | 核心局限 |
|---|---|---|
| ChatGPT / DeepSeek | 自然对话、知识广博 | 线性输出，信息零散难沉淀 |
| XMind / MindMaster | 结构清晰、适合复习 | 静态编辑，无法与 AI 协作 |
| Perplexity / 秘塔 | 实时搜索聚合 | 输出为文本列表，非图谱形态 |

**底部高亮**：
> **市场缺口**：缺乏将"AI 动态生成"与"思维导图结构化沉淀"深度融合的工具。

**演讲备注**：
> "我们分析了市场上三类主要产品。AI 对话工具擅长生成，但不擅长组织；思维导图工具擅长组织，但不擅长生成；AI 搜索工具虽然能聚合信息，但输出形态仍然是文本列表，而不是一张图。这就留下了一个明确的市场缺口。"

---

## 第 4 页：解决方案

**标题**：我们的方案 — 对话即构图

**要点**：
- 每一次 AI 对话 = 思维导图上的一个节点
- AI 主动推荐延伸方向 → 用户点击 → 自动创建子节点
- 支线聊完后自动生成摘要
- 整张地图 = 结构化的知识资产

**配图建议**：
- 一张简化的产品界面示意图，展示根节点 → 分支节点的结构

**演讲备注**：
> "我们的解决方案非常简单直观：把每一次 AI 对话封装成思维导图上的一个节点。当 AI 回答完你的问题后，它会主动推荐几个延伸方向，比如你正在学 Transformer，它会问你想深入了解自注意力机制、位置编码还是多头注意力。你点击一个，系统自动创建一个子节点，你就在这个子节点里继续聊，而主线完全不受影响。"
> "聊完之后，这个子节点会自动生成一句摘要。最后你得到的是一张完整的知识地图，而不是一段散乱的对话。"

## 第 5 页：产品演示（核心功能）

**标题**：核心功能一览

**要点**（图标 + 简短说明，6 个功能排成 2 行 3 列）：

1. **节点即对话** — 每个节点承载完整 AI 对话历史
2. **非线性分支** — AI 推荐方向，点击自动深入
3. **节点摘要** — 支线结论自动提炼
4. **多工作区** — 按主题/项目组织知识
5. **文件上传** — AI 基于文档内容生成节点
6. **中英双语** — 完整国际化支持

**演讲备注**：
> "我快速过一下产品的六大核心功能。首先是节点即对话，每个节点里面都可以和 AI 多轮对话；第二是非线性分支，AI 会主动给你推荐下一步该往哪走；第三是自动摘要，让你不用点开每个节点也知道里面讲了什么；第四是多工作区，你可以按课程、按项目来组织不同的地图；第五是文件上传，你可以丢一篇论文进去，AI 帮你提炼成思维导图；最后是中英双语，界面可以随时切换。"

## 第 7 页：技术架构

**标题**：全栈 TypeScript 技术架构

**要点**（架构图 + 技术栈表格）：

**架构简图**：
```
Web 浏览器 / Android App / Admin 后台
         ↓
   Node.js + Express（主服务 + Admin 服务）
         ↓
  MongoDB（文档） + Neo4j（图谱） + Redis（缓存）
```

**核心技术栈**：

| 前端 | 后端 | 数据层 |
|---|---|---|
| React 18 + Vite | Node.js + Express | MongoDB 6.x |
| TypeScript 5.x | TypeScript 5.x | Neo4j 5.x |
| Tailwind CSS | OpenAI SDK | Redis 5.x |
| @xyflow/react | Multer / pdf-parse | 极光推送 |
| Capacitor 8 | Vitest 测试 | |

**演讲备注**：
> "技术架构上，我们采用全栈 TypeScript，保证前后端类型一致。前端基于 React + Vite，画布引擎使用 @xyflow/react，移动端用 Capacitor 封装。后端是 Node.js + Express，数据库采用 MongoDB 存文档、Neo4j 存图谱关系、Redis 做缓存和限流。整个项目严格执行代码规范，核心业务逻辑的单元测试覆盖率达到 80% 以上。"

## 第 8 页：创新点与差异化

**标题**：我们有什么不同？

**要点**（对比表格，突出优势）：

**vs 线性 AI 对话**：
- 信息组织：从"线性时间轴"升级为"图谱化结构"
- 分支探索：从"打断上下文"升级为"子节点独立对话"
- 知识沉淀：从"导出文本"升级为"导出思维导图"

**vs 传统思维导图**：
- 内容生成：从"手动输入"升级为"AI 自动生成"
- 动态性：从"静态文档"升级为"动态生长"

**核心创新总结**：
1. 首创"对话即构图"的交互范式
2. AI 从"回答者"升级为"导航员"，主动推荐分支
3. 自动摘要让地图越用越清晰

**演讲备注**：
> "这里我想强调我们的三个核心创新。第一，我们首创了'对话即构图'的交互范式，这不是在思维导图里嵌入一个聊天窗口，而是让对话的自然分支直接映射为地图的拓扑结构。"
> "第二，AI 的角色从单纯的回答者变成了导航员，它会主动告诉你接下来可以往哪几个方向深入。"
> "第三，自动摘要机制让这张地图越用越清晰，而不是越用越臃肿。"

## 附录：PPT 制作建议

### 视觉风格
- **主色调**：深蓝 + 白色（与产品界面风格一致）
- **字体**：标题用思源黑体 Bold，正文用思源黑体 Regular
- **图表**：多用对比表格、架构简图，少用大段文字

### 动画建议
- 第 2 页痛点：用"信息淹没"的动画效果（文字逐渐模糊/堆叠）
- 第 4 页方案：用节点逐个出现的动画，模拟地图生长
- 第 5-6 页功能：图标逐个飞入

### 演讲节奏
- 痛点 + 市场：2 分钟
- 方案 + 功能演示：3 分钟
- 技术架构 + 创新点：2 分钟
- 应用场景 + 成果 + 规划：2 分钟
- 结尾：1 分钟
