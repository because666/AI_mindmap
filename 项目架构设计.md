## 二、前端架构（client）

### 2.1 技术栈

| 层级 | 技术 |
|---|---|
| 框架 | React 18 + TypeScript |
| 构建 | Vite 6 |
| 状态管理 | Zustand |
| 路由 | React Router v7 |
| 样式 | Tailwind CSS 3 |
| 图标 | Lucide React |
| 画布 | @xyflow/react |
| 国际化 | i18next + react-i18next |
| Markdown | react-markdown + remark-gfm |
| 移动端 | Capacitor 8 |
| 测试 | Vitest |

### 2.2 目录结构

```text
client/src/
├── components/
│   ├── Canvas/           # 思维导图画布
│   ├── Chat/             # AI 对话面板
│   ├── Layout/           # 页面布局
│   ├── Node/             # 节点渲染
│   ├── Search/           # 搜索面板
│   ├── Settings/         # 设置面板
│   ├── Onboarding/       # 新手引导
│   └── Workspace/        # 工作区面板
├── stores/               # Zustand 全局状态
├── services/             # API 调用封装
├── hooks/                # 自定义 Hook
├── locales/              # i18n 翻译资源
├── types/                # TypeScript 类型
├── utils/                # 工具函数
└── data/                 # 静态数据配置
```

### 2.3 核心状态

- `appStore`：当前工作区、地图、选中节点、画布视图
- `chatStore`：当前节点对话历史、输入状态
- `workspaceStore`：工作区列表、消息、公告
- `settingsStore`：用户设置、API Key、语言、主题

### 2.4 关键交互流程

```text
用户点击节点
    ↓
appStore.selectNode(nodeId)
    ↓
chatStore 加载该节点对话历史
    ↓
用户发送消息
    ↓
services/aiService.chat(nodeId, messages)
    ↓
服务端流式返回 AI 回答
    ↓
前端解析回答（含工具调用、延伸方向）
    ↓
更新对话历史 / 创建节点 / 渲染延伸方向按钮
```

## 三、后端架构（server）

### 3.1 技术栈

| 层级 | 技术 |
|---|---|
| 框架 | Express.js + TypeScript |
| 数据库 | MongoDB |
| 图数据库 | Neo4j（可选） |
| 缓存 | Redis（可选，默认 MongoDB 会话） |
| AI 调用 | axios + 各厂商 SDK |
| 文件上传 | Multer |
| 文档解析 | pdf-parse / mammoth / xlsx |
| 定时任务 | node-cron |
| 邮件 | nodemailer |
| 测试 | Vitest |

### 3.2 目录结构

```text
server/src/
├── config/               # 配置、prompt 模板
├── routes/               # API 路由
├── services/             # 业务服务
├── middleware/           # 中间件（认证、限流、日志）
├── models/               # 数据模型
├── utils/                # 工具函数
├── types/                # TypeScript 类型
└── data/                 # 数据库连接
```

### 3.3 核心服务

| 服务 | 职责 |
|---|---|
| aiService | 调用 AI 模型，处理流式输出与工具调用 |
| toolExecutor | 执行 AI 返回的工具调用（创建节点等） |
| nodeService | 节点 CRUD 与节点关系管理 |
| conversationService | 对话历史管理 |
| workspaceService | 工作区与地图管理 |
| fileService | 文件上传与解析 |
| authService | 用户认证与会话 |

### 3.4 AI 服务设计

```text
用户请求
    ↓
aiService 组装 messages + system prompt + tools
    ↓
调用模型（GLM / DeepSeek / OpenAI / 自定义）
    ↓
流式解析返回内容
    ↓
识别工具调用 → toolExecutor 执行
    ↓
返回文本 + 工具执行结果
```

System prompt 已要求 AI 在回答末尾输出 `🌱 延伸方向：`，供前端解析为可点击按钮。

## 四、数据库设计

### 4.1 MongoDB 集合

| 集合 | 说明 |
|---|---|
| users | 用户账号、配置、状态 |
| workspaces | 工作区元数据 |
| maps | 地图元数据，归属工作区 |
| nodes | 节点数据（标题、内容、摘要、位置、父节点等） |
| conversations | 节点对话历史 |
| messages | 工作区消息与公告 |
| feedbacks | 用户反馈 |
| auditLogs | 审计日志 |
| events（规划中） | 埋点事件 |
| modelConfigs（规划中） | 内置 AI 模型配置 |

### 4.2 节点模型关键字段

```typescript
interface Node {
  _id: string;
  mapId: string;
  parentIds: string[];       // 支持多父节点
  title: string;
  content: string;
  summary?: string;          // 节点摘要（新增）
  position: { x: number; y: number };
  color?: string;
  collapsed?: boolean;
  createdAt: Date;
  updatedAt: Date;
}
```

### 4.3 Neo4j（可选）

用于存储复杂的节点关系网络，支持多对多关系查询。当前阶段主要依赖 MongoDB 的 `parentIds` 和 `relations` 字段。

## 五、后台管理架构（admin）

### 5.1 结构

```text
admin/
├── client/               # Admin 前端（React）
│   ├── src/pages/        # 各管理页面
│   └── package.json
└── server/               # Admin 服务（Express）
    ├── src/routes/       # Admin API
    └── package.json
```

### 5.2 功能模块

- Dashboard：数据统计
- Users：用户管理、封禁
- Workspaces：工作区管理、置顶
- AIUsage：AI 用量统计
- Feedback：用户反馈
- Announcements：公告管理
- Push：消息推送
- AuditLogs：审计日志
- AdminAccounts：管理员账号

### 5.3 规划中增强

- Dashboard 2.0：漏斗、留存、实时监控
- 用户洞察中心：活跃度分层、价值排序
- 内容与地图运营：地图浏览器、模板工厂
- AI 成本与质量监控：按用户/地图/功能分摊
- 系统健康监控：API 延迟、错误率、告警

## 七、部署架构

### 7.1 本地开发

```powershell
npm install
npm run dev:server   # http://localhost:3001
npm run dev:client   # http://localhost:5173
```

### 7.2 Docker 部署

```text
docker-compose up -d
```

启动 MongoDB、Neo4j、Redis（可选）与服务端。

### 7.3 生产部署

```text
1. 配置 .env
2. npm run build:server
3. npm run build:client
4. 部署静态文件到 Web 服务器
5. 使用 PM2 启动服务端
```

参考 `.trae/skills/server-deploy-checklist/SKILL.md`。

## 八、安全设计

- 用户密码 bcrypt 加密
- JWT 认证，配合 refresh token
- API 限流（express-rate-limit）
- Helmet 安全头
- 文件上传类型与大小限制
- 敏感信息通过环境变量读取，禁止硬编码
- 审计日志记录关键操作

## 九、扩展方向

- **实时协作**：引入 Socket.io 或 Yjs
- **向量检索**：为节点内容生成 embedding，支持语义搜索
- **插件化模型**：通过后台配置接入更多 AI 提供商
- **公开地图分享**：生成只读链接
- **社区市场**：用户共享模板与优质地图
